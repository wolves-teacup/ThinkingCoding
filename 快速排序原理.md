# 快速排序（Quick Sort）原理详解

## 一、什么是快速排序？

**快速排序**（Quick Sort）是由英国计算机科学家 Tony Hoare 于1959年提出的一种高效的排序算法。它采用**分治思想**，将一个大问题分解为若干个小问题来解决。

> **核心思想**：选择一个"基准"元素，将数组分为两部分——左边都比基准小，右边都比基准大，然后递归地对左右两部分进行同样的操作。

---

## 二、算法原理

### 分治策略

快速排序使用分治法（Divide and Conquer）策略，分为三个步骤：

1. **分解**：从数组中选一个元素作为基准（pivot），将数组分为两个子数组，使得左子数组所有元素 ≤ 基准 ≤ 右子数组所有元素
2. **解决**：递归地对左右子数组进行快速排序
3. **合并**：因为子数组是原地排序的，不需要合并操作

### 动图演示

```
初始数组: [38, 27, 43, 3, 9, 82, 10]

第1轮: 选基准=10
  [3, 9, 10, 43, 27, 38, 82]
  左子数组: [3, 9]    右子数组: [43, 27, 38, 82]

第2轮: 左子数组选基准=9 → [3, 9]
        右子数组选基准=82 → [43, 27, 38, 82]
          左子数组: [43, 27, 38]  右子数组: []

第3轮: [43, 27, 38] 选基准=38 → [27, 38, 43]

最终结果: [3, 9, 10, 27, 38, 43, 82]
```

### 分区过程详解

以 Lomuto 分区方案为例（选择最后一个元素为基准）：

```
数组: [38, 27, 43, 3, 9, 82, 10]  基准=10

   i=-1  j=0: 38 > 10, 不交换
   i=-1  j=1: 27 > 10, 不交换
   i=-1  j=2: 43 > 10, 不交换
   i=-1  j=3: 3 ≤ 10, i=0, 交换 arr[0]和arr[3] → [3, 27, 43, 38, 9, 82, 10]
   i=0   j=4: 9 ≤ 10, i=1, 交换 arr[1]和arr[4] → [3, 9, 43, 38, 27, 82, 10]
   i=1   j=5: 82 > 10, 不交换

   最后: 交换 arr[2]和arr[6] → [3, 9, 10, 38, 27, 82, 43]
   
   基准10的最终位置: 索引2
```

## 三、时间复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| **最好情况** | O(n log n) | 每次分区都能将数组均匀分成两半 |
| **最坏情况** | O(n²) | 数组已经有序或逆序，且每次都选到最小/最大元素作为基准 |
| **平均情况** | O(n log n) | 随机排列的数组 |

### 空间复杂度

- **平均情况**：O(log n) —— 递归调用的栈深度
- **最坏情况**：O(n) —— 递归深度为 n

> 快速排序是**原地排序**（in-place），不需要额外的数组空间，但需要递归栈空间。

---

## 四、稳定性

**快速排序是不稳定的排序算法。**

> 举例：数组 `[5a, 3, 5b, 2]`，选基准为 3
> 分区后：`[2, 3, 5b, 5a]`
> 两个 5 的相对顺序发生了变化（5a 跑到了 5b 后面）

---

## 五、优化方案

### 1. 三数取中法（Median of Three）

不再固定选择第一个或最后一个元素，而是从 `low`、`mid`、`high` 三个位置中取中值作为基准，可以有效避免最坏情况。

```
arr = [1, 2, 3, 4, 5, 6, 7, 8, 9]
low=0 → 1, mid=4 → 5, high=8 → 9
中值 = 5，选 5 作为基准
```

### 2. 随机选择基准

随机选择一个元素作为基准，从概率上避免最坏情况。

### 3. 小数组使用插入排序

当子数组长度小于某个阈值（如 10~20）时，改用插入排序，因为对于小数组，插入排序比快速排序更快。

```java
if (high - low < 10) {
    insertionSort(arr, low, high);
    return;
}
```

### 4. 三路快速排序（3-Way Quick Sort）

将数组分为三部分：小于基准、等于基准、大于基准。适合处理包含大量重复元素的数组。

```
[小于基准 | 等于基准 | 大于基准]
```

## 六、优缺点

### 优点

- **平均性能极好**，O(n log n) 的时间复杂度
- **原地排序**，空间复杂度低
- **缓存友好**，数据访问具有局部性
- 实际应用中通常比其他 O(n log n) 算法更快

### 缺点

- **不稳定**，相等元素的相对位置可能改变
- **最坏情况**时间复杂度为 O(n²)
- 对递归深度敏感，极端情况下可能导致栈溢出
- 对于小规模数据，不如插入排序高效

---

## 七、适用场景

- 大规模数据的排序（n > 1000）
- 数据随机分布
- 对稳定性没有要求
- 需要高效排序的通用场景

> 快速排序是许多编程语言标准库中的默认排序算法（如 Java 的 `Arrays.sort()` 对基本类型使用双轴快速排序，C++ 的 `std::sort` 使用内省排序——快速排序+堆排序的混合体）。

---

## 八、与其他排序算法的比较

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定 |
|------|---------|---------|------|------|
| **快速排序** | O(n log n) | O(n²) | O(log n) | ❌ |
| **归并排序** | O(n log n) | O(n log n) | O(n) | ✅ |
| **堆排序** | O(n log n) | O(n log n) | O(1) | ❌ |
| **冒泡排序** | O(n²) | O(n²) | O(1) | ✅ |

> **总结**：快速排序是实际应用中最快的通用排序算法之一，虽然存在最坏情况 O(n²) 的理论缺陷，但通过良好的优化策略（三数取中、随机化等），在实际使用中几乎不会遇到最坏情况。
