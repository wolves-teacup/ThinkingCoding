# 冒泡排序（Bubble Sort）原理详解

## 一、什么是冒泡排序？

**冒泡排序**是一种简单的排序算法。它重复地遍历要排序的数组，依次比较相邻的两个元素，如果它们的顺序错误（如前一个比后一个大），就交换它们。遍历数组的工作会重复进行，直到没有需要交换的元素为止。

> **名称由来**：每一轮排序中，最大的元素会像"气泡"一样慢慢"浮"到数组的末端。

---

## 二、算法原理

### 核心思想

1. 比较相邻的元素。如果第一个比第二个大，就交换它们。
2. 对每一对相邻元素做同样的工作，从开始第一对到结尾的最后一对。这步做完后，最后的元素会是最大的数。
3. 针对所有的元素重复以上的步骤，除了最后一个。
4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

### 动图演示

```
初始数组: [64, 34, 25, 12, 22, 11, 90]

第1轮: 比较 6 次，将最大值 90 放到末尾
  [64, 34, 25, 12, 22, 11, 90]  → 比较交换...
  → [34, 25, 12, 22, 11, 64, 90]

第2轮: 比较 5 次，将次大值 64 放到倒数第二
  [34, 25, 12, 22, 11, 64, 90]
  → [25, 12, 22, 11, 34, 64, 90]

... 以此类推 ...

最终结果: [11, 12, 22, 25, 34, 64, 90]
```

## 三、时间复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| **最好情况** | O(n) | 数组已经有序，优化版只需遍历一次 |
| **最坏情况** | O(n²) | 数组完全逆序，需要 n×(n-1)/2 次比较 |
| **平均情况** | O(n²) | 随机排列的数组 |

### 空间复杂度

- **O(1)** —— 只使用了常数个额外变量（原地排序）

---

## 四、稳定性

**冒泡排序是稳定的排序算法。**

> **稳定性**：如果两个相等的元素在排序前后的相对位置不变，则称该排序算法是稳定的。

冒泡排序中，只有当 `arr[j] > arr[j+1]` 时才交换，相等时不交换，因此相等元素的相对顺序不会改变。

---

## 五、优化方案

### 1. 提前结束标志

如果某一轮遍历中没有发生任何交换，说明数组已经有序，可以直接结束排序。

```java
boolean swapped = false;
for (int j = 0; j < n - 1 - i; j++) {
    if (arr[j] > arr[j + 1]) {
        // 交换
        swapped = true;
    }
}
if (!swapped) break; // 没有交换，提前结束
```

### 2. 记录最后交换位置

每轮排序后，记录最后一次发生交换的位置，该位置之后的元素已经有序，下一轮无需再比较。

---

## 六、优缺点

### 优点

- 实现简单，代码易于理解
- 稳定性好
- 空间复杂度低，是原地排序
- 对于基本有序的数组，优化后效率较高

### 缺点

- 时间复杂度高，平均和最坏都是 O(n²)
- 数据量大时效率极低
- 不适合大规模数据排序

---

## 七、适用场景

- 数据量很小（如 n < 100）
- 数据基本有序
- 教学演示排序算法的基本原理
- 对性能要求不高的简单场景

> 在实际开发中，对于大规模数据，通常使用快速排序、归并排序或堆排序等更高效的算法。
